Default

API

<nuri-nav-item> takes a text label as its child and an onpress handler. It builds its internal structure by cloning a native <template> (NOT lit-html / JSX / imperative DOM · decision 52) and forwards the press contract to the composed <nuri-list-interactive-item>. An optional <nuri-list-item-leading> child is hoisted into the row as a start book-end; the trailing caret is always auto-filled.

PropTypeDefaultDescription
onpress handler — Forwarded to the composed interactive wrapper, which fires a bubbling press on click / Enter / Space. Nav-agnostic — routing is the author's handler.
children text — The label. Wrapped in the .nuri-type-md--em SSOT utility (md em-weight). The trailing caret is auto-filled — the author never writes it.
<nuri-list-item-leading> slot — Optional. An authored leading book-end (e.g. an <nuri-icon-avatar>) is hoisted into the composed row before the label — list-item positions it as the start book-end by element type (decision 51). Everything else stays the label.

Recipe, not variant

The disclosure pattern could have been a variant="nav" on <nuri-list-item> — but that path swells one component with use-case branches. A recipe instead NAMES the composition and leaves the primitives single-purpose (decision 52). NavItem expands to: 

<nuri-nav-item onpress="goTo('/card')">Activate card</nuri-nav-item>

  →

<nuri-list-interactive-item onpress="goTo('/card')">
  <nuri-list-item>
    <span class="nuri-type-md--em">Activate card</span>
    <nuri-list-item-trailing>
      <nuri-icon name="caret-right" size="md"></nuri-icon>
    </nuri-list-item-trailing>
  </nuri-list-item>
</nuri-list-interactive-item>

The caret is muted with no muted prop on Icon (decision 38): the trailing book-end sets color: var(--nuri-border-strong), and <nuri-icon> paints in currentColor, so the caret reads muted by inheritance — the same trick IconAvatar's subtle variant uses.

Live demos

Press fires a bubbling press event (wired here to a console log to show the nav-agnostic contract); on press the row fills with a flat bg-subtle wash. Row height is the parent List's density — a List feature, not a NavItem one.

Token mapping

NavItem is skip-emit — it has no @layer tokens block and ships no build/components/nav-item.ts. Every value it renders is already a component token on the primitive it composes (List · Base token mapping). The only CSS the recipe adds is the muted-caret colour and the display: contents host collapse:

Part / state CSS property DTCG CSS var Reference
trailing caret color chrome.border-strong — var(--nuri-border-strong) · muted via currentColor (decision 38)
label font-weight type.md-em-weight — .nuri-type-md--em · SSOT type utility
