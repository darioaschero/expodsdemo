Composition & RN mapping

RN map: <ScrollView style={{ flex: 1 }}>. Scroll is a thin component over ScrollView — the grow + overflow region between the pinned chrome of a Screen. The web side renders an inner <div> with flex:1; min-block-size:0; overflow-y:auto inside a display:contents wrapper.

Why a primitive, not a Box prop. In RN, scrolling is a component (ScrollView), not a View style — there is no overflow:scroll on a View that maps cleanly. So overflow can never be a Box prop (it would have no View analogue · R1 parity). Scroll is its own primitive precisely so the web ↔ RN seam stays 1:1.

Padding goes INSIDE, on a Box. The Scroll box itself takes no padding — in RN, padding on a ScrollView belongs to contentContainerStyle, never the ScrollView box. So the web pattern places a <nuri-box padding> as the scroll's child, which maps directly to contentContainerStyle: 

<nuri-scroll>                       <!-- ScrollView style={{flex:1}} -->
  <nuri-box padding="lg">            <!-- contentContainerStyle padding -->
    <nuri-stack gap="lg">…</nuri-stack>
  </nuri-box>
</nuri-scroll>

See Screen for the column that hosts Scroll, and the My vault playground for the full scaffold rendered live inside the device frame.

API

Scroll takes no layout props — its fill + block-axis overflow are fixed structure. The only prop is the web-only host override.

Prop Type Default Description
as string "div" Host element on the web side. Has no RN analogue — RN always renders <ScrollView>.

Anatomy & parts

Scroll is a single-part component. The web side renders an inner <div> (or the element named by as) inside the display:contents wrapper; the RN side renders a <ScrollView> with style={{flex:1}}. Padding for the content is NOT owned here — it lives on a child Box (contentContainerStyle).

Part Element (web) Element (RN) Owns
root div (or as) ScrollView {flex:1} flex fill + block-axis overflow (flex:1 · min-block-size:0 · overflow-y:auto · overscroll-behavior:contain)

Structure · no tokens

Like Screen, Scroll dereferences no design tokens. Its fill + overflow are pure structure — flex:1; min-block-size:0; overflow-y:auto; overscroll-behavior:contain — not design-scale values. So scroll.css carries an empty @layer tokens block per decision 37, and the pipeline emit step skips writing build/components/scroll.ts (decision 52 skip-emit · same posture as Stack / Box / Screen).
