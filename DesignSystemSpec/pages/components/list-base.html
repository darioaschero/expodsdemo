Default

API

<nuri-list> is the vertical container; <nuri-list-item> is the presentational row; <nuri-list-interactive-item> is the optional pressable wrapper that composes around a row. The row's appearance is a function of which positions the author fills, not a variant prop (decision 51). The List has no gap prop — inter-row rhythm is each row's density-driven min-height plus author-placed <nuri-separator>s.

<nuri-list> · container

PropTypeDefaultDescription
density "sm" | "md" | "lg" md Sets the row min-height for every row child — sm 60px (size-xl), md 72px (size-2xl), lg 90px (size-3xl). A container decision projected onto the rows: the selector targets nuri-list-item only, so author-placed <nuri-separator>s stay 1px across every density. Now EMITted as component tokens (list.densitySmMinHeight etc.) for machine-checked RN parity (decision 52).

<nuri-list-item> · presentational row

PropTypeDefaultDescription
none — — No props. The row is purely presentational (role="listitem") — it carries NO interactivity (decision 52). Appearance is set by which book-end positions the author fills. The row has no inline padding — content is edge-to-edge so it aligns with the full-width separators; the container supplies the outer margin. The two fixed spacing values (block guard, inter-part gutter) are EMITted as component tokens (listItem.paddingBlock, listItem.gap).

<nuri-list-interactive-item> · pressable wrapper

PropTypeDefaultDescription
onpress handler — Wraps its child <nuri-list-item> in an inner role="button" tabindex="0" element so the content IS the accessible name (Enter / Space activation, a bubbling press event). It is nav-agnostic — it only fires a press; routing is the author's handler. This is the N+7 a11y fix: no copied aria-label, no double-read (decision 52).

<nuri-list-item-leading> / <nuri-list-item-trailing> · book-end markers

PropTypeDefaultDescription
none — — No props. Presence is the API: an authored wrapper becomes an auto-width book-end (leading before the content, trailing after) in document order — content is NOT reparented within the row (decision 51). Omit the wrapper entirely for an empty side.

Anatomy & parts

A row is a flex row in document order: an optional auto-width leading book-end, the stretched content (flex:1), and an optional auto-width trailing book-end. When interactive, the wrapper adds an action element that WRAPS the whole row — its flat press wash bleeds past the row edges via a counter-margin (full-bleed by counter-margin · revised N+8.1). The row has no inline padding, so content sits flush with the full-width separators; the container supplies the outer margin (decision 52).

Live demos

One row shape. The presentational row carries no press — fill the content and an optional trailing book-end. The interactive row wraps the same shape in <nuri-list-interactive-item>; on press the action box fills with a flat bg-subtle wash, made full-bleed by a counter-margin (revised N+8.1).

Density

density on the container sets every row's min-height — sm 60px, md 72px (default), lg 90px. Author-placed <nuri-separator>s stay 1px at every density. The same simple list, three densities.

Roadmap

  • spec in migration-test
    React Native implementation

    The RN List / ListItem / InteractiveListItem translation (density → row min-height via context; a Pressable wrapping the presentational row; all fixed values dereferencing the emitted tokens via resolveToken) lives in the migration test. Promotion to a permanent lib/ home tracks the shared RN-spec-home question in roadmap/index.md.

  • deferred · cross-platform-gated
    Interactive control in a trailing book-end

    The settings-row-with-Switch pattern (a real interactive control nested in a row) is deferred: an interactive row's press target and a nested control's hit area conflict, and the resolution differs web ↔ RN. Revisit when a cross-platform nested-hit-area model is settled.
