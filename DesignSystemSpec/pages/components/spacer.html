Composition & RN mapping

Spacer has three modes, all dispatched in CSS off its attributes (it is behaviour-free — no JS reflection):

  • Grow (no size) → flex: 1. Absorbs the free space of the parent flex container on its main axis. Drop one between two siblings to push them apart, or before/after content to shove it to an edge.
  • Fixed (size) → a definite gap from the semantic space scale (inline-size in a row, block-size in a column). A deliberate gap without reaching for the parent's gap.
  • Proportional (grow) → flex-grow: n. A grow spacer at grow="2" takes twice the share of a default (grow="1"); two grow spacers at 1 / 2 split the slack 1:2. Ignored when size is set.

RN map. A grow spacer is the RN/SwiftUI Spacer idiom — a <View style={{ flex: 1 }} /> (and flex: n for grow="n", since a positive flex is proportional flexGrow). A fixed spacer is a <View> with a definite width (row) or height (column) from tokens.space. 

grow (default)   <View style={{ flex: 1 }} />
grow="2"         <View style={{ flex: 2 }} />          // proportional flexGrow
size="lg" · row  <View style={{ width:  tokens.space.lg }} />
size="lg" · col  <View style={{ height: tokens.space.lg }} />

See the My vault playground for grow + proportional spacers pushing the total and the fund actions down a filled Scroll body.

API

Three props, all prop-driven CSS. grow applies only to a grow spacer (it is ignored when size is set).

Prop Type Default Description
direction "row" | "column" "row" The axis a fixed size fills (row → inline-size, column → block-size). For the grow default it is declarative — flex:1 grows on the parent's main axis either way.
size "xs" | "sm" | "md" | "lg" | "xl" unset (grow) FIXED extent over the semantic space scale. Omitted → the spacer GROWS (flex:1) instead.
grow "1" | "2" | "3" | "4" "1" Proportional grow share for a GROW spacer — maps literally to flex-grow. Two grow spacers at 1/2 split the free space 1:2. Ignored when size is set. (Renamed from weight · 2026-06-01 · font-weight collision.)

Anatomy & parts

Spacer is a single-part component. Unlike Stack / Box it sizes the host directly — no inner element, no display:contents wrapper (the same shape as Separator). The web side renders a <div>; the RN side renders a <View>.

Part Element (web) Element (RN) Owns
root div View grow (flex / flex-grow) or a fixed inline/block extent on the chosen axis

Live demos

Each spacer is invisible — the visible witnesses are the bounded outlined containers and the buttons inside them.

Token mapping

Prop value → CSS property → DTCG → CSS var → reference. Only size dereferences a semantic token; grow (the default and the proportional prop) dispatches to flex literals. Layout primitives have no @layer tokens block per decision 37 — the dispatch happens in @layer rules through attribute selectors (nuri-spacer[size="lg"] { inline-size: var(--nuri-space-lg); }).
