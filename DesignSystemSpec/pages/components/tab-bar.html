Default

API

A compound of two elements. web ↔ RN is 1:1 on props — both <nuri-tab-bar> / <TabBar> and <nuri-tab-bar-item> / <TabBar.Item> carry the same props. Note the absence of a variant or size prop (decision 56 · single appearance, icon-only items).

<nuri-tab-bar> · controller

Prop Type Default Description
value string (a child item's value) first item's value The selected destination — the single source of truth. Defaults to the first <nuri-tab-bar-item>'s value when omitted. Tapping an item writes its value here; the controller then toggles active on each child (F-SELECTED-VALUE-1).
label string none An accessible name for the navigation landmark — set as aria-label on the built <nav>. Recommended when a screen has more than one nav region.

<nuri-tab-bar-item> · destination

Prop Type Default Description
value string required This destination's identity. Matched against the controller's value to determine the selected item.
name string (icon registry key) required The glyph for this destination — an <nuri-icon> registry key (e.g. vault, coin-vertical, clock). Rendered at size="md"; the selected item flips it to the filled weight (Icon weight coupling · decision 38).
label string derived from name The accessible name for this icon-only target (F-ARIA-LABEL-1). When omitted it is derived from the kebab name (coin-vertical → "coin vertical"); set it explicitly for a human-readable destination name ("Coin"). The inner <nuri-icon> is decorative.

Anatomy & parts

TabBar is a four-part compound. The tab-bar controller owns the selected value and builds a <nav> row; each item is a native <button> that composes a single <nuri-icon>. The selected item carries aria-current="page" — destination semantics, not tablist semantics (F-TABBAR-ROLE-1).

Part Element Owns Description
tab-bar nuri-tab-bar the selected value, click delegation The controller. display: contents — invisible in layout; it builds the nav row on mount and reparents the items into it.
nav nav the bar surface + height, item layout A native <nav aria-label> landmark. A flex row at --nuri-tab-bar-height (the one emitted token), bg-canvas, items stretched full-height and equal-width.
item button destination target, rest/selected/pressed visuals A native <button>, equal-width (flex: 1), full bar height. data-active + aria-current reflect selection; aria-label is its accessible name.
icon nuri-icon the glyph + its weight A composed <nuri-icon size="md">. Decorative (aria-hidden). The selected item sets fill for the filled weight; the rest render regular.

States

An item is in one of three states (operator-locked · N+9, zero new tokens). not selected recedes in border-strong with a regular glyph; selected (the controller's value match) reads in text-primary with a filled glyph; pressed is the tactile squash (scale(0.97)) plus a colour shift to text-muted on :active — press and hold an item to see it. Pressed has no background change and does not alter the glyph weight; it is transient and applies to whichever item is held, selected or not. No hover — TabBar is mobile-first.

Theming · chrome-only

TabBar is chrome-only: the active accent has no effect. The selected item reads in text-primary (not accent-solid like a Tab) — a navigation bar is persistent chrome and stays neutral so it never competes with accent-driven content on the screen (decision 56). What does re-resolve is the theme: bg-canvas + text-primary + border-strong are chrome tokens, so a [data-theme] scope repaints the whole bar with no extra rules.

Token mapping

Part × CSS property → DTCG → CSS var → reference. TabBar bakes exactly one structural decision — the bar height — into tab-bar.ts (EMIT · decision 52). Everything else is direct-semantic consumption: the item treatment reads the shared chrome + interaction vocabulary directly in @layer rules (the IconAvatar / Topbar precedent · decisions 46/50), so there are no per-item component tokens. — means web-only / direct reference.

Behavioural delta · web ↔ RN

Props are 1:1; behaviour is budgeted per-component (RISKS R1). TabBar adds the role delta specific to a destination switcher and inherits the shared compound-state, icon-name, and interaction deltas. This is the migration agent's third surface alongside anatomy and token mapping.

Friction Surface Web React Native Resolution
F-TABBAR-ROLE-1 navigation role A <nav aria-label> landmark of native <button> destinations; the selected button carries aria-current="page". Correct for a destination switcher without presupposing a router, and distinct from Tabs' role="tablist" (which describes an in-page panel switcher). RN has no accessibilityRole that maps 1:1 to web's <nav> / aria-current pairing for a destination bar. It approximates with accessibilityRole="tab" + accessibilityState={{ selected }} inside an accessibilityRole="tablist" container — the closest available shape, which over-claims tablist semantics. Budgeted as F-TABBAR-ROLE-1 (RISKS R1). Web keeps the correct nav/aria-current model; RN uses the tab/selected approximation until a closer role exists. Props + selection behaviour stay 1:1.
F-SELECTED-VALUE-1 selected value DOM-driven: the controller holds value as an attribute, delegates taps, and toggles active on each child item. Shared state flows one direction through the DOM. State-driven: value is a controlled prop on <TabBar>; selection flows via onValueChange and React.cloneElement injects active + onSelect into items. Inherited from Tabs (decision 43). Same prop + compound shape; the shared-state mechanism differs (DOM attribute toggling vs cloneElement injection). See TabBarDemo in the migration test.
F-ARIA-LABEL-1 accessible name Each item is icon-only, so it needs an accessible name: explicit label wins, else it is derived from the kebab name. Set as aria-label on the inner button. Same rule via accessibilityLabel on the Pressable. Inherited from IconButton (decision 38). The derivation is identical on both sides.
F-PRESSED-1 pressed :active → transform: scale(0.97) + colour shift to text-muted Pressable render-prop style={({ pressed }) => …} Inherited interaction model. Same constant (--nuri-interaction-press-scale), different trigger mechanism.
F-FOCUS-1 focus-visible :focus-visible → 2px solid var(--nuri-focus-ring) No DOM focus model; the accessibility role substitutes Inherited. Ring is web-only by design.

    Web ships the correct <nav> / aria-current destination model now; RN approximates with tab/selected. Revisit when RN exposes a closer navigation role, or when the consuming app's router lands and a <a href> / link-based item becomes appropriate. See RISKS.
