Default

API

A compound of two elements. web ↔ RN is 1:1 — both <nuri-tabs> / <Tabs> and <nuri-tab> / <Tabs.Tab> carry the same props. Note the absence of a variant or size prop (decision 43 · single appearance, button-sm option shape).

<nuri-tabs> · controller

Prop Type Default Description
value string (a child tab's value) first tab's value The selected option — the single source of truth. Defaults to the first <nuri-tab>'s value when omitted. Clicking a tab writes its value here; the controller then toggles active on each child (F-SELECTED-VALUE-1).

<nuri-tab> · option

Prop Type Default Description
value string required This option's identity. Matched against the controller's value to determine the active tab. The tab's text content is its visible label.
disabled boolean false Makes the option non-selectable. The controller's click handler ignores disabled tabs.

Anatomy & parts

Tabs is a three-part compound. The tabs controller owns the selected value; the list is a composed <nuri-box> that supplies the surface (background + radius + inset padding); each tab is a native <button role="tab"> in a flex row inside that Box.

One Two
Part Element Owns Description
tabs nuri-tabs the selected value, click delegation The controller. display: contents — invisible in layout; it composes the list + Box on mount.
list nuri-box → div[role=tablist] surface (bg + radius + inset), flex rhythm A composed <nuri-box background="strong" radius="md" padding="xs"> wrapping a flex role="tablist". The gap is --nuri-tabs-gap.
tab button option shape, rest/active visuals A native <button role="tab"> in button-sm shape. Equal-width (flex: 1). aria-selected + data-active reflect selection.

States

A tab is in one of three states. rest is transparent with muted text; active (the controller's value match) fills with accent-solid; disabled is non-selectable. pressed is the tactile squash (scale(0.97)) on :active — press and hold a tab to see it. No hover — Tabs is mobile-first.

Behavioural delta · web ↔ RN

Props are 1:1; behaviour is budgeted per-component (RISKS R1). Tabs adds two deltas specific to a compound select control and inherits the two shared interaction deltas. This is the migration agent's third surface alongside anatomy and token mapping.

Friction Surface Web React Native Resolution
F-SELECTED-VALUE-1 selected value DOM-driven: the controller holds value as an attribute, delegates clicks, and toggles active on each child tab. Shared state flows one direction through the DOM. State-driven: value is a controlled prop on <Tabs>; selection flows via onValueChange and React.cloneElement injects active + onSelect into children. Same prop + compound shape; the shared-state mechanism differs (DOM attribute toggling vs cloneElement injection). Pipeline emits a controlled RN compound — see TabsDemo in the migration test.
F-KEYBOARD-NAV-1 roving focus Baseline only: the active tab is tabIndex 0, the rest -1; each activates on native Enter/Space. Full arrow-key roving across the tablist is deferred this session. No DOM tab order; selection is touch-driven. Arrow-key roving has no mobile analogue — the deferral is web-only debt. Deferred (F-KEYBOARD-NAV-1). The role="tablist" + aria-selected + roving tabIndex baseline is in place; arrow-key handling lands in a later session. See RISKS.
F-PRESSED-1 pressed :active → transform: scale(0.97) Pressable render-prop style={({ pressed }) => …} Inherited interaction model. Same token (tab.pressScale), different trigger mechanism.
F-FOCUS-1 focus-visible :focus-visible → 2px solid var(--nuri-focus-ring) No DOM focus model; accessibilityRole="tab" substitutes Inherited. Ring is web-only by design.
