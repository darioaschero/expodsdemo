Default

API

Separator takes one prop, y-space (amendment 49.1 · N+8.1) — the vertical breathing room above and below the line, over the semantic space scale (default sm, accepts none for a flush hairline). It is a margin-block, NOT a thicker fill: the visible hairline stays exactly 1px at every value. Thickness, orientation, and inset remain deliberately out of scope (decision 49). web ↔ RN parity is 1:1 — the web side dispatches purely in CSS (nuri-separator[y-space="lg"], no JS reflection); the RN side renders a 1px View with marginVertical: space[ySpace] (see docs/migration-tests/button-matrix/index.tsx).

Prop Type Default Description
y-space none | xs | sm | md | lg | xl sm Vertical breathing room above and below the line, dispatched over the semantic space scale via margin-block (the line stays 1px). none (= 0) is a flush hairline. This is the List family's inter-row rhythm now that List is gap-free (decision 51 · 52). The host is exposed as role="separator" with aria-orientation="horizontal" — the ARIA analogue of <hr>.

Anatomy & parts

Separator is a single-part component. Unlike Stack/Box (invisible display:contents wrappers), the <nuri-separator> host is the visible line — it paints directly, with no inner element. The RN side renders a 1px <View>.

Live demos

Separator's whole job is to sit between rows. These show it closing the old Stack divider question (decision 49): a divider is just a Separator the author drops between children, not a Stack prop.

Token mapping

CSS property → DTCG → CSS var → reference. Separator dereferences the hairline colour plus the y-space margin — both semantic tokens, no component-token aliasing. There is no @layer tokens block per decision 49; the references happen directly in @layer rules (background: var(--nuri-border-subtle); margin-block: var(--nuri-space-sm)), so the colour re-resolves per theme and y-space dispatches the space scale via attribute selector — with no component token in between. Separator therefore stays skip-emit (no build/components/separator.ts).
