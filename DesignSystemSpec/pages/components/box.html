API

The props signature. Padding precedence: edge-specific (padding-start, padding-end, padding-top, padding-bottom) wins over axis (padding-x, padding-y) which wins over uniform (padding). Mirrors the CSS selector ordering.

Prop Type Default Description
padding "xs" | "sm" | "md" | "lg" | "xl" unset Uniform inset on all four sides. Maps to --nuri-space-{xs..xl}.
padding-x "xs" | "sm" | "md" | "lg" | "xl" unset Inline-axis inset (logical · RTL-aware via padding-inline). Overrides padding on the inline axis.
padding-y "xs" | "sm" | "md" | "lg" | "xl" unset Block-axis inset (padding-block). Overrides padding on the block axis.
padding-start "xs" | "sm" | "md" | "lg" | "xl" unset Single edge · inline-start (left in LTR; right in RTL). Overrides axis + uniform.
padding-end "xs" | "sm" | "md" | "lg" | "xl" unset Single edge · inline-end. RTL-aware.
padding-top "xs" | "sm" | "md" | "lg" | "xl" unset Single edge · block-start.
padding-bottom "xs" | "sm" | "md" | "lg" | "xl" unset Single edge · block-end.
background "canvas" | "subtle" | "strong" | "accent-solid" | "accent-subtle" unset Surface fill. canvas/subtle/strong read the theme-only chrome bg tokens; the accent-* pair reads the accent × theme tokens. accent-solid also sets text colour to --nuri-accent-on-solid so labels stay legible on the fill. An unknown value warns to the console and paints nothing.
radius "sm" | "md" | "lg" | "full" unset Corner softness. Maps to --nuri-radius-{sm..full}. full is the 9999px pill/circle sentinel. Unknown values warn and apply no radius.
center boolean false Inline-axis auto margins (margin-inline: auto). Only meaningful when the host has a bounded width; no-op otherwise.
fill boolean false Grow to fill the main axis of a flex parent and become a flex column, so a filling child (e.g. <nuri-stack fill>) fills in turn. The web analogue of RN's ScrollView contentContainerStyle={{ flexGrow: 1 }} — pairs with Scroll + Spacer to push trailing content to the bottom. Uses flex: 1 0 auto so content taller than the viewport scrolls instead of clipping (decision 60).
as string "div" Host element on the web side. Has no RN analogue — RN always renders <View>.

Anatomy & parts

Box is a single-part component. The web side renders an inner <div> (or the element named by as) inside the display:contents wrapper; the RN side renders a <View>.

Live demos

Selected examples covering the three padding modes (uniform, axis, edge) and the centering helper. Box itself is invisible — the content inside is the visible witness to the inset.

Surface · background & radius

background and radius (decision 42) extend Box from pure-layout to a paint-bearing surface — the evidence-gate opened by Tabs, the first real consumer (its tablist surface is a composed Box). Chrome backgrounds (canvas / subtle / strong) stay neutral across accents; the accent-* pair tracks the active accent. accent-solid also flips text to --nuri-accent-on-solid.

Token mapping

Prop value → CSS property → DTCG → CSS var → reference. Every padding leaf dereferences a space token; only center dispatches to a CSS literal. Layout primitives have no @layer tokens block per decision 37 — the dispatch happens in @layer rules through attribute selectors (.nuri-box[data-padding="md"] { padding: var(--nuri-space-md); }).

Roadmap

  • shipped · N+6.5
    background prop

    Promoted in N+6.5 (decision 42). The evidence-gate cleared when Tabs needed a chrome-filled track surface — the first real consumer. Vocabulary as scoped: chrome (canvas / subtle / strong) + accent (accent-solid / accent-subtle). See the Surface section.

  • shipped · N+6.5
    radius prop

    Promoted alongside background (decision 42) — the same Tabs surface needed rounded corners. Accepts semantic radius leaves (sm · md · lg · full).

  • shipped · N+11
    fill prop

    Promoted in N+11 (decision 60). The evidence-gate cleared when the My-vault scroll body needed to fill the viewport so a Spacer could push the fund actions to the bottom — the web analogue of RN's ScrollView contentContainerStyle={{ flexGrow: 1 }}. Pairs with Stack fill and Scroll.

  • planned · evidence-gated
    max-width prop

    Requires a NEW semantic vocabulary (page-shaped rem values, different from semantic.size's element-shaped px values). Defer until a page-layout component surfaces the need.

  • planned
    RN spec long-term home

    Today the Box RN definition lives in the migration-test pair as a local module. As of N+6.5 n≥2 component RN specs co-habit that file (Box, Button, IconButton-pending, Switch, Tabs) — the long-term-home question (peer lib/components/box/box.tsx? a lib/spec/ namespace?) is now live. Tracked in roadmap/index.md.
