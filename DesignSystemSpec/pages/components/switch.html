API

The props signature. web ↔ RN is 1:1 — every prop below exists on both the <nuri-switch> custom element and the RN <Switch> component, with the same name and accepted values. Note the absence of a size or variant prop (decision 44 · single size, single appearance).

Prop Type Default Description
checked boolean false (off) The toggle state. Reflected to aria-checked on the inner <button role="switch">. Clicking toggles the host attribute — the DOM is the source of truth (F-CHECKED-STATE-1).
disabled boolean false Disables interaction. Visual: opacity 0.4 + cursor: not-allowed + pointer-events: none (belt & suspenders).
accent "lilac" | "neutral" inherited Self-scope override (Tier 2). Writes data-accent on the inner <button>. Only affects the ON track (which paints accent-solid); the OFF track is chrome-only and ignores it.

Anatomy & parts

Switch is a two-part component: the track is the root <button role="switch"> pill that owns size, shape, and the OFF/ON background; the knob is the circular slider that translates along the inline axis by a derived travel distance.

Part Element Owns Description
track button size, shape, OFF/ON background, all state visuals The 60×36 pill. Carries role="switch" + aria-checked. OFF → bg-inverse-muted, ON → accent-solid.
knob span the sliding disc (28×28) Always bg-canvas so it lifts off either track colour. Translates by travel (24px = track − knob − 2·inset) when ON; squashes to scale(0.92) while pressed.

States

Two binary axes: checked (off / on) and disabled. All shown live. pressed is the knob squash (scale(0.92)) on :active — press and hold any live switch to see it. No hover — Switch is mobile-first.

Behavioural delta · web ↔ RN

Props are 1:1; behaviour is budgeted per-component (RISKS R1). Switch adds one delta specific to a toggle control (F-CHECKED-STATE-1) and inherits the three shared interaction deltas. This is the migration agent's third surface alongside anatomy and token mapping.

Friction Surface Web React Native Resolution
F-CHECKED-STATE-1 checked state DOM-driven: click toggles the host checked attribute, attributeChangedCallback re-syncs aria-checked. The DOM attribute is the source of truth (uncontrolled by default). State-driven: checked is a controlled prop; the consumer holds useState and passes onValueChange. No DOM to reflect into. Same prop name + accepted values; the source-of-truth mechanism differs (DOM attribute vs React state). The pipeline emits a controlled RN component — see SwitchDemo in the migration test.
F-PRESSED-1 pressed :active → knob transform: scale(0.92) Pressable render-prop style={({ pressed }) => …} applied to the knob Inherited interaction model. Same token (knobPressScale), different trigger mechanism.
F-FOCUS-1 focus-visible :focus-visible → 2px solid var(--nuri-focus-ring) No DOM focus model; accessibilityRole="switch" substitutes Inherited. Ring is web-only by design.
F-DISABLED-1 disabled opacity 0.4 + cursor: not-allowed + pointer-events: none disabled prop → same opacity; no cursor concept Inherited. Opacity 1:1; cursor dropped on RN.
