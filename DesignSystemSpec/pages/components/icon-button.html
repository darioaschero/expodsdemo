API

The props signature. web ↔ RN is 1:1 — every prop below exists on both the <nuri-icon-button> custom element and (when implemented) the RN <IconButton> component, with the same name and accepted values. Note the absence of a size prop (decision 40 · single size).

Prop Type Default Description
name string (registry key) required Kebab-case glyph name from the icons.js registry. Forwarded to the inner <nuri-icon size="md">.
variant "solid" | "soft" | "ghost" "soft" Visual treatment. solid fills with the active accent, soft uses neutral chrome, ghost is transparent at rest (chrome-only).
accent "lilac" | "neutral" inherited Self-scope override (Tier 2). Writes data-accent on the inner element. Only affects solid; soft and ghost ignore it.
label string derived from name Accessible name (aria-label). An icon-only control has no text, so a label is required (F-ARIA-LABEL-1). Omit to auto-derive from the kebab name (arrows-down-up → "arrows down up").
disabled boolean false Disables interaction. Visual: opacity 0.4 + cursor: not-allowed.

Anatomy & parts

IconButton is a two-part component: the circular root surface owns size, shape, and all state visuals; the icon part is the wrapped <nuri-icon> glyph, which inherits colour via currentColor.

Part Element Owns Description
root button size, shape, background, all state visuals The 48px circular surface. Carries the aria-label.
icon nuri-icon the glyph (md weight) A single <nuri-icon size="md">, aria-hidden, inheriting the root's color.

Variants

Three variants, mirroring Button (decision 39 · ghost is the shared cross-component variant). solid carries accent identity; soft and ghost are chrome-only. ghost is transparent until pressed, never bordered. Default is soft.

States

Three states. default and disabled are shown live; pressed is forced statically (its live form is :active — press and hold any live button to see it). No hover — IconButton is mobile-first.

Behavioural delta · web ↔ RN

Props are 1:1; behaviour is budgeted per-component (RISKS R1). IconButton inherits Button's three deltas and adds two specific to an icon-only, always-circular control. This is the migration agent's third surface alongside anatomy and token mapping.

Friction Surface Web React Native Resolution
F-PRESSED-1 pressed :active + transform: scale(0.97) + bg-pressed swap Pressable render-prop style={({ pressed }) => …} Inherited from Button. Same tokens, different trigger mechanism.
F-FOCUS-1 focus-visible :focus-visible → 2px solid var(--nuri-focus-ring) No DOM focus model; accessibilityRole="button" substitutes Inherited from Button. Ring is web-only by design.
F-DISABLED-1 disabled opacity 0.4 + cursor: not-allowed + pointer-events: none disabled prop → same opacity; no cursor concept Inherited from Button. Opacity 1:1; cursor dropped on RN.
F-ARIA-LABEL-1 accessible name Icon-only → no text node. aria-label on the inner <button>, auto-derived from name unless label given; inner <svg> is aria-hidden accessibilityLabel prop on Pressable, same derive-from-name fallback; glyph importantForAccessibility="no" Mechanism differs (aria-label vs accessibilityLabel) but the derive-from-name contract is 1:1. Pipeline emits the RN attribute.
F-TOUCH-TARGET-1 touch target 48px circle is the hit area; a 28px glyph sits centred with a ~10px optical ring. CSS box = touch box 48px clears the 44pt iOS / 48dp Android floor without hitSlop; smaller glyph-only controls would need explicit hitSlop Single 48px size (decision 40) is chosen precisely so no hitSlop shim is ever needed. The container size IS the accessibility guarantee.
