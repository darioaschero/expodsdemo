Default

API

A single element. <nuri-typography-stack> is the flex container — its only prop is the structural direction author value, read directly from the attribute by CSS (no data-*, because it is static, not JS-computed). It owns the inter-line rhythm (the gap is not parameterised — that is what distinguishes it from <nuri-stack>). The lines are plain <nuri-typography>; their size / emphasis / muted carry the hierarchy (decision 53 eliminated the former <nuri-typography-stack-element> and its level sub-scale; the author composes the props per line).

<nuri-typography-stack> · container

Prop Type Default Description
direction "column" | "row" column Stacking axis. column gives a tight vertical leading (space.2xs gap); row lays the lines out inline with align-items: baseline and a wider space.xs gutter so mixed-size lines share a text baseline. CSS reads the attribute directly. The gap is owned, not a prop.

Lines · <nuri-typography>

The stack wraps plain <nuri-typography> lines — it adds no line element of its own. Each line picks its size + emphasis (the metric SSOT) and an optional muted boolean for the supporting tone. A recommended size/emphasis/muted pairing per hierarchy step is deliberately not shipped — that doctrine is invented advice until the type-scale principles are documented (P11 · decision 53), so the author composes the props directly for now.

Decision 53 deliberately keeps the hierarchy on <nuri-typography>'s own props rather than a stack- scoped level: a single primitive with one type vocabulary beats a parallel sub-scale that collides with size. The muted prop is a boolean, not a tone enum (no tone="primary|muted|accent" — P11).

Anatomy & parts

TypographyStack is a single-element container that composes <nuri-typography> lines. Unlike Topbar there is no region routing (the lines are homogeneous), so the container does no reparenting — it is the flex box, the lines are its flex items. No Shadow DOM, no <slot> (decision 46 lineage). The line is not a part the stack owns — it is the composed Typography primitive (decision 53).

Live demos

The two axes against the shapes that motivated the family — transaction rows, summary lines. column is the default primary→muted vertical pairing; row lays lines inline on a shared baseline (a label + value pair).

Theming · text tones

A default line inherits currentColor (the cascade's primary); a muted line paints --nuri-text-muted — a chrome (theme-only, accent-invariant) token. Toggling theme re-resolves the muted tone with no extra rules; the muted supporting line is the one that visibly shifts in dark.

Token mapping

The stack owns only the rhythm; the lines own their type. TypographyStack has no @layer tokens block per decision 47 / decision 53. It dispatches one property — the inter-line gap, per direction. The lines' font size / line-height / weight ride the .nuri-type-* utility the composed <nuri-typography> applies (SSOT in styles/typography.css), and the supporting-line colour is the muted dispatch on Typography itself (nuri-typography[data-muted] { color: var(--nuri-text-muted); }).
