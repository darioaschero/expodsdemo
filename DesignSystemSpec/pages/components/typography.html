API

Prop Type Default Description
size "xs" | "sm" | "md" | "lg" | "xl" | "3xl" "md" Type scale step. Each step maps to a fixed font-size · line-height · letter-spacing tuple from the foundation (--nuri-type-{step}-*). Unknown values warn and fall back to md. 2xl is intentionally reserved (gap between xl and 3xl).
emphasis boolean absent Presence flips weight from regular to semibold (across all leaves). Same visual jump as Markdown **bold** emphasis — never bold (700), always semibold (600) post-N+6.5 polish.
muted boolean absent Presence dims the text colour to --nuri-text-muted for a supporting / secondary line (timestamp, helper text); absent inherits the cascade primary via currentColor. A boolean, not a tone enum — there is deliberately no tone="primary|muted|accent" (the same restraint Icon takes · decision 38 · P11). Reflected as data-muted; CSS owns the colour (decision 42 attribute-dispatch). The colour re-resolves per [data-theme], so the muted line shifts in dark mode. Added decision 53 · N+8.2.
align "start" | "center" | "end" absent Block-aligns the text via text-align (logical · RTL-aware). The host is display:inline by default, so align flips it to display:block so the alignment takes effect and the box fills its container's inline size — e.g. a flex-column item stretches full-width, so end right-aligns the text. Pure attribute-dispatch CSS (no JS reflection · survives the size/emphasis class rewrite). RN maps to <Text style={{ textAlign }}> (decision 59).

Sizes

Six leaves of the type ramp · same scale as the typography foundation. Each demo card stacks the three prop axes: the regular weight, the emphasis weight (semibold), and the muted tone (text-muted · for supporting lines).

Token mapping

Typography owns no component tokens. The component applies an existing utility class on the host (.nuri-type-{size} or .nuri-type-{size}--em) which in turn references the --nuri-type-{step}-* primitives. Single source of truth for type rules lives in styles/typography.css · values in tokens-primitive.css.

Roadmap

  • 2xl size unlock Intentional gap between xl (30) and 3xl (57). Land when a concrete consumer surfaces — likely ~40/48.
  • as semantic-tag prop Optional polymorphic prop (as="p|h1|h2|…") to render the host as a semantic element. Defer until accessibility-evidence trigger materializes.
  • color / tone enum — deferred The supporting-line tone shipped as the muted boolean (decision 53): present → --nuri-text-muted, absent → currentColor primary. A broader tone="primary|muted|accent" enum stays deliberately deferred — adding it before a real third-tone consumer is the P11 floodgate Icon refused (decision 38). It lands only when a consumer needs a tone muted can't express.
