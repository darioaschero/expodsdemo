API

The props signature. web ↔ RN is 1:1 — every prop below exists on both the <nuri-button> custom element and (when implemented) the RN <Button> component, with the same name and accepted values.

Prop Type Default Description
variant "solid" | "soft" | "ghost" "soft" Visual treatment. solid fills with the active accent, soft uses neutral chrome, ghost is transparent at rest (chrome-only).
accent "lilac" | "neutral" inherited Self-scope override (Tier 2). Writes data-accent on the inner element. Only affects solid; soft ignores it.
size "lg" | "md" | "sm" "md" Container scale. lg 60px (soft mobile primary) · md 48px (inline action · default) · sm 36px (badge / dense action). Typography + radius couple asymmetrically — see Sizes.
disabled boolean false Disables interaction. Visual: opacity 0.4 + cursor: not-allowed.

Variants

Three variants. solid carries accent identity (changes colour with the active accent); soft and ghost are chrome-only (stay neutral regardless of accent). soft is the canonical "quiet alternative"; ghost is the tertiary, zero-weight-at-rest action — transparent until pressed, never bordered. Default is soft.

Sizes

Three sizes. lg (60px) is the soft mobile primary; md (48px) is the inline action and the default; sm (36px) is the dense / badge action. The couplings are asymmetric (decision 41): font-size breaks at the md/sm boundary (lg + md share type-md; sm drops to type-sm), while radius breaks at the lg/md boundary (lg = radius-md; md + sm = radius-sm).

States

Three states for v0. default and disabled are shown live; pressed is forced statically (its live form is :active — click and hold any live button to see the same visual). No hover — Button is mobile-first.

Behavioural delta · web ↔ RN

Props are 1:1; behaviour is budgeted per-component (RISKS R1). Button's budget is three deltas. Each names the web mechanism, the RN mechanism, and what the migration pipeline emits — the third machine-readable surface alongside anatomy and token mapping.

Friction Surface Web React Native Resolution
F-PRESSED-1 pressed :active pseudo-class + transform: scale(0.97) + bg-pressed swap Pressable render-prop style={({ pressed }) => …} — explicit onPressIn/Out, scroll-cancellation differs Pipeline emits the render-prop signature; same tokens, different trigger mechanism.
F-FOCUS-1 focus-visible :focus-visible → 2px solid var(--nuri-focus-ring) outline No DOM focus model. accessibilityRole="button" + accessibilityState substitute the behaviour; visual ring intentionally absent Ring is web-only by design; RN side ships the a11y role, not the outline.
F-DISABLED-1 disabled [disabled] → opacity 0.4 + cursor: not-allowed + pointer-events: none disabled prop → same opacity; no cursor concept; touch suppressed by Pressable disabled Opacity is 1:1; cursor is web-only and dropped on RN.
