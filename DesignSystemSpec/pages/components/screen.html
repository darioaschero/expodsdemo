Composition & RN mapping

RN map: <View style={{ flex: 1 }}> (later the themed SafeAreaProvider root). Screen is a thin component over a full-height View — the flex:1 column that fills the navigator's content area. The web side renders an inner <div> (or the element named by as) inside a display:contents wrapper.

Screen is NOT the navigator. The thing that holds Screen + the bottom TabBar and owns the safe-area is the navigator (React Navigation in RN; the device frame in the playground) — framework chrome, not an authored View. So the TabBar is a SIBLING of Screen, never a child: in RN the tab bar is rendered once at navigator level, persistent across screens. Screen stays inset-agnostic — no env() / inset props (safe-area is handled in ONE place upstream, decision 58). 

<!-- Navigator chrome (device frame · React Navigation) owns the safe-area -->
<nuri-screen>
  <nuri-topbar>…</nuri-topbar>        <!-- pinned · natural height -->
  <nuri-scroll>                       <!-- flex:1 · grows + scrolls -->
    <nuri-box padding="lg">
      <nuri-stack gap="lg">…</nuri-stack>
    </nuri-box>
  </nuri-scroll>
</nuri-screen>
<nuri-tab-bar>…</nuri-tab-bar>          <!-- SIBLING · navigator chrome -->

See the My vault playground for the scaffold rendered live inside the device frame, and Scroll for the growing body.

API

Screen takes no layout props — its full-height column is fixed structure. The only prop is the web-only host override.

Prop Type Default Description
as string "div" Host element on the web side ("section", "main", …). Has no RN analogue — RN always renders <View>.

Anatomy & parts

Screen is a single-part component. The web side renders an inner <div> (or the element named by as) inside the display:contents wrapper; the RN side renders a <View> with flex:1.

Part Element (web) Element (RN) Owns
root div (or as) View {flex:1} full-height flex column (flex-direction:column · flex:1 · min-block-size:0)

Structure · no tokens

Unlike Stack (gap → space) or Box (padding → space, background → chrome), Screen dereferences no design tokens. Its fill is pure structure — display:flex; flex-direction:column; flex:1; min-block-size:0 — not a design-scale value. So screen.css carries an empty @layer tokens block per decision 37, and the pipeline emit step skips writing build/components/screen.ts (decision 52 skip-emit · same posture as Stack / Box / Scroll).
