Default

API

A compound of three elements. <nuri-topbar> is the shell; <nuri-topbar-start> and <nuri-topbar-end> are the named side-region markers — everything else inside the shell is the centre. The only prop is the structural center boolean; leading / trailing occupancy is detected, not configured (it flips the edge padding from CSS · decision 42). Edge padding has a smart default — empty edge lg, occupied edge sm, centred bar xs — that the optional inset / inset-start / inset-end override only when the author needs to (decision 46.1).

<nuri-topbar> · shell

Prop Type Default Description
center boolean false Switches the layout from left-aligned chrome (centre absorbs the slack, left-justified) to an equal-side centred action bar: the two side regions share the slack equally (flex: 1) so the centre stays optically centred regardless of leading / trailing widths. Reflected as data-center on the host.
inset · inset-start · inset-end "xs" | "sm" | "lg" auto Overrides the edge padding. The default is computed from structure — empty edge lg, occupied edge sm, centred bar xs. inset is the symmetric shorthand; inset-start / inset-end set one edge and win over it. Reflected as data-inset-start / data-inset-end; the value pick lives in CSS (decision 46.1). Asymmetric insets on a center bar shift the optical centre — keep them symmetric there.

<nuri-topbar-start> / <nuri-topbar-end> · region markers

Prop Type Default Description
none — — No props. Presence + content of the marker is the API: an occupied start / end sets data-leading="filled" / data-trailing="filled" on the shell (else "empty"), which flips the edge padding. Omit the wrapper entirely for an empty edge.

Anatomy & parts

Topbar is a three-region shell. On mount the topbar element reparents its authored children into three built region containers — the named <nuri-topbar-start> / <nuri-topbar-end> wrappers go to the side regions; everything else goes to the centre. No Shadow DOM, no <slot> (the Tabs light-DOM lineage · decision 43).

Live demos

The structural range: a left-aligned title with a double trailing affordance, a filled close button (the IconButton fill passthrough · decision 40.1), a back-chevron leading row, a centred <nuri-tabs> switch, and the center action bar. The title stays put while the trailing region stacks as many icon buttons as it needs.

Theming · chrome surface

The shell paints --nuri-bg-canvas with --nuri-text-primary — both chrome (theme-only) tokens. Toggling theme repaints the whole bar with no extra rules; the accent surfaces only through the composed controls (e.g. a solid Button in the centre bar), never the shell itself.

Token mapping

CSS property → DTCG → CSS var → reference. Topbar has no @layer tokens block per decision 46 — every value dereferences a semantic token directly in @layer rules through the host element selector and data-* attribute dispatch (e.g. nuri-topbar:not([data-center])[data-leading="filled"] { padding-inline-start: var(--nuri-space-sm); }). — means a CSS literal, not a token.
