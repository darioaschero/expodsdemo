Build assets manifest

The RN side consumes five machine-generated files. All are emitted by npm run build from pipeline/tokens-parser.js (orchestrator) which runs the parser slices in pipeline/parsers/ (semantic-cascade + per-component CSS walk). Pipeline sources live in pipeline/ hand-maintained; outputs in build/ are 100% generated and never hand-edited (decision 35 · N+6.0.4).

Tokens JSON (DTCG)
build/tokens.json 216 colour primitives · DTCG-nested · { $type, $value } per leaf · themed scales 4-deep, alphas 3-deep · W3C standard, ingestible by any DTCG-aware tool
Tokens TS (runtime sets only)
build/tokens.ts 38 runtime TokenPath leaves across nested + flat sets: the context-dependent chrome: Record<Theme, {…13 leaves}> + accent: Record<Accent, Record<Theme, {…6 leaves}>>, PLUS the cascade-invariant dimension singletons space (8) · size (7) · radius (4) — these travel as flat runtime sets too, returned context-free by resolveToken (dimensions promoted to runtime at N+6.1 / N+6.1.1). So it is not only context-dependent vocabulary that travels (decision 34 · N+6.0.3). The type scale ships as a separate directly-accessed namespace (decision 54). Per-component numerics moved to per-component files; the prior buttonBase constants block is retired.
Component tokens TS (per component)
build/components/<name>.ts One file per component — 8 today: button · icon-button · switch · tabs · list · list-item · list-interactive-item · tab-bar (layout primitives with an empty @layer tokens skip-emit). References to pipeline-inlined primitive sets resolve to literal numerics / strings at build (e.g. pressScale: 0.97 from var(--nuri-interaction-press-scale)); references to runtime sets — now including the dimension fields — emit as '<group>.<leaf>' as const satisfies TokenPath (e.g. lgMinHeight: 'size.xl', solidBg: 'accent.solid'). The consumer dereferences TokenPath strings against the live (accent × theme) runtime slice via a resolveToken(tokens, path) helper.
TokenPath union
build/token-paths.ts Discriminated string union over every runtime-set leaf. Mechanically derived from classifyAll; per-component files import type { TokenPath } and TS enforces leaf-name agreement at compile time. Renaming a runtime leaf without re-emitting the union fails the TS compile.
Icon registry TS
build/icons.ts The shared glyph registry emitted once as a typed module: an IconName union + IconWeight + icons: Record<IconName, Record<IconWeight, string>> (17 glyphs × 3 weights). RN renders the verbatim SVG path string through react-native-svg's SvgXml — one registry, two readers (decision 48 · N+6.8).
Shape contract
Each runtime export's nesting depth equals the dimensions its source CSS vars span across [data-<dim>=…] selectors (decision 28 · classify-by-cascade). Adding a new cascade dimension auto-extends the shape; unmapped signatures fail the build. Group names (chrome / accent) are stable consumer API — don't rename them in app code. The SET_POLICY registry at pipeline/parsers/semantic.js routes each set between runtime emit and pipeline-inline emit (decision 34).

Cascade → Context mapping

The web side uses the CSS cascade via data-* attributes; the RN side uses a single React Context with merge-on-override semantics (decision 27). The translation is attribute → field on a single context value, not Provider-per-dimension.

Web (Nuri) RN (NuriThemeContext) Notes
data-theme="light"|"dark" on <html> or any wrapper mode field on NuriThemeContext Default 'light'. Web internal name is theme for the attribute; the public RN prop is mode (1:1 with web prop on <nuri-scope>).
data-accent="lilac"|"neutral"|… accent field on NuriThemeContext Default 'lilac'. Accent type is exported from build/tokens.ts.
<nuri-scope accent="…" mode="…"> <NuriScope accent="…" mode="…"> (composite) One Provider wraps the subtree; reads ambient context, spreads overrides. Per-dimension Providers explicitly rejected — see RISKS R1 F-SCOPE-1.
<nuri-button accent="neutral"> (Tier 2) <Button accent="neutral"> prop Same name, different mechanism. Web writes data-accent on the inner element; RN reads useContext(NuriThemeContext) with accent prop as override.
data-density · data-neutral density · neutral fields Auto-discoverable once their CSS cascade blocks land (decision 28). Reserved — not NuriThemeValue fields today; they join when their web tokens exist (P11).
data-font="ios|android|pixel" — (no counterpart) Web-only emulation overlay for design exploration. RN inherits the platform's system font stack natively; no font field on NuriThemeContext (amendment 27.1).

NuriScope composite · spec

Start here for migration

The <nuri-scope> page consolidates the web cascade mechanism, this RN NuriThemeContext spec, and the cascade↔context delta in one place. It is the canonical web↔RN scope reference; this section is the condensed spec.

The RN counterpart to <nuri-scope> is orthogonal, not a cross-product theme registry. Each dimension is one entry in a single NuriThemeContext with merge-on-override semantics; consumers spread additional dimensions onto a child <NuriScope> without enumerating themes. Pre-computed (∏ dimensions) theme tuples are explicitly rejected (8 accent × 3 font × 2 density × 7 neutral = 336 themes is the roadmap reductio).

Context
NuriThemeContext single React Context carrying one entry per dimension
Merge semantics
<NuriScope accent="neutral"> reads ambient via useContext and emits { ...ambient, accent: 'neutral' }. Unspecified dimensions inherit; specified ones win. No dimension is "all-or-nothing".
Props (1:1 with web, minus font)
mode accent density neutral same names, same value sets as the web <nuri-scope> attributes. font exists on web for emulation overlay only; not migrated (amendment 27.1).
Component consumption
Single const { accent, mode, ... } = useContext(NuriThemeContext) then destructure. No per-dimension Provider chain.
Unistyles posture
Consumed for stylesheet ergonomics only if used at all. Its theme registry is bypassed — runtime values come from NuriThemeContext, not UnistylesRuntime.setTheme(...) (which would have to enumerate ∏ dims tuples).

  mode: 'light',
  accent: 'lilac',

const NuriScope: React.FC<Partial<NuriThemeValue> & { children: React.ReactNode }> = ({

  ...overrides

Tier 2 self-scope pattern · per-component accent prop

Leaf components (Button, Tag, Chip, IconButton — P5) take an optional accent prop that overrides the ambient context for the component itself only. The pattern: read ambient via useContext, override with the prop using nullish coalescing, then use the result to index into accent[a][t].

Web mechanism
Writes data-accent on the inner element. CSS cascade re-resolves the component-token var() chain at that element (decision 9 · component-token selector pattern).
RN mechanism
useContext(NuriThemeContext) for ambient · ?? fallback for the prop · resulting accent + mode index into the nested accent export from build/tokens.ts.

Note the size-coupled geometry + label type (decision 41 · 55): per-size minHeight/paddingX/radius resolve inline through the runtime sets, and the label's typeStyle key tracks size — there is no styles.label StyleSheet entry:

// Per-size geometry triple + size-coupled label type (decision 41 · 55).
const GEOMETRY: Record<ButtonSize, { minHeight: TokenPath; paddingX: TokenPath; radius: TokenPath }> = {
  lg: { minHeight: button.lgMinHeight, paddingX: button.lgPaddingX, radius: button.lgRadius },
  md: { minHeight: button.mdMinHeight, paddingX: button.mdPaddingX, radius: button.mdRadius },
  sm: { minHeight: button.smMinHeight, paddingX: button.smPaddingX, radius: button.smRadius },
};
const LABEL_KEY: Record<ButtonSize, TypeKey> = { lg: 'mdEm', md: 'mdEm', sm: 'smEm' };

const Button: React.FC<ButtonProps> = ({
  variant = 'soft',
  accent: accentProp,
  size: sizeProp = 'md',
  disabled,
  onPress,
  children,
}) => {
  const { mode, accent: ambientAccent } = React.useContext(NuriThemeContext);
  const accent: Accent = accentProp ?? ambientAccent;
  const tokens: RuntimeTokens = { chrome: chrome[mode], accent: accentTokens[accent][mode], space, size, radius };
  const geometry = GEOMETRY[sizeProp];
  return (
    <Pressable
      onPress={onPress}
      disabled={disabled}
      accessibilityRole="button"
      accessibilityState={{ disabled: !!disabled }}
      style={({ pressed }) => [
        styles.base,
        {
          minHeight:         resolveToken(tokens, geometry.minHeight) as number,
          paddingHorizontal: resolveToken(tokens, geometry.paddingX)  as number,
          borderRadius:      resolveToken(tokens, geometry.radius)    as number,
        },
        variantStyle(variant, accent, mode, pressed),
        pressed && !disabled && { transform: [{ scale: button.pressScale }] },
        disabled && { opacity: button.disabledOpacity },
      ]}
    >
      <Text style={[typeStyle(LABEL_KEY[sizeProp]), { color: labelColor(variant, accent, mode) }]}>
        {children}
      </Text>
    </Pressable>
  );
};
