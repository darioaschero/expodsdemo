How to read these tables

Set · core

Each row is one primitive token. DTCG is the dot-notation name used by the planned tokens.json pipeline; CSS var is the custom property you reference in stylesheets; Value is the literal hex (or rgba for alpha); Type is the DTCG $type inferred from the prefix (see AGENTS.md).

Each non-alpha scale exposes 24 tokens — 12 steps in both -light and -dark. At this layer there is no "mode": gray-1-light and gray-1-dark are two distinct primitives, addressable in parallel. The semantic layer is where one or the other gets picked for a given data-theme.

Step roles · Radix 12-step functional model

Each step has the same role across every scale, regardless of theme. The roles apply to both the light and dark variants.

Radix names some steps after web pseudo-classes ("UI element hover", "Solid hover"). Nuri is mobile-first and reads these steps as pressed instead — same visual purpose, different trigger. See Button for the Nuri-side state model.

Neutral · active scale

The semantic layer references neutrals through the --nuri-color-neutral-* aliases — 24 tokens that point at the currently active scale (driven by data-neutral on <html>). Switch the active scale via the Neutral selector in the topbar; the Reference column below shows which underlying primitive each alias resolves to. Compare the six candidate scales side-by-side in Colour · Exploration.

Brand · Lilac

Single brand scale (Radix custom from #beaaff). Steps 9 and 10 have identical values in light and dark — the brand stays the brand. That's the property the semantic layer uses to freeze --nuri-accent-solid (step 9) and --nuri-accent-solid-pressed (step 10) across themes.

Alpha · Black & White

Theme-invariant alpha scales (0.05 → 0.95 opacity). Used for overlays, dividers on coloured backgrounds, glass surfaces, and any place a tinted darken / lighten is needed without resolving to a specific neutral hue. No -light/-dark suffix.

Roadmap

    No status primitives ship today. The full Radix 12-step scales (Jade / Amber / Red / Blue) land in the same session as the first status-using component (Toast · Alert · StatusPill); the foundation page grows a Status group at that point. No placeholders pre-shipped.

    Pick one of the seven candidates (see Exploration) and remove the others — the data-neutral attribute will disappear with them.
