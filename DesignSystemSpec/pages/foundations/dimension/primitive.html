Approach

The dimension primitive layer uses direct-pixel naming: each token's value equals its name. --nuri-px-12 is literally 12 px. No indexed scale, no T-shirt mnemonics — the vocabulary IS the value (decision 32).

Twelve values ship. Each one exists because at least one consumer reaches for it — either a semantic dimension leaf (spacing · sizing · radius) or a chrome surface (the docs shell consumes --nuri-px-48 directly for the 44 px touch-target retrofit). New primitives arrive when a new consumer surfaces; speculative additions are caught at the test boundary (decision 30 · P11).

Hard rule 1 / P3: components do NOT consume these primitives directly. Component CSS reads from the semantic dimension layer (--nuri-space-*, --nuri-size-*, --nuri-radius-*); the semantic layer is the only direct consumer of the primitive vocabulary. See Button for the worked example.
