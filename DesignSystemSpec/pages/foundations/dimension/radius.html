Approach

Radius is the vocabulary border-radius consumes. Roles read as "md corner softness" rather than "12 pixels" — the same role-vs-literal split that Spacing and Sizing apply (see decision 36 and amendment 36.1).

Four leaves ship — a smaller scale than spacing or sizing because corner softness has a narrower useful range. Three chain through the --nuri-px-N primitive layer (decision 32): radius.sm = 6, radius.md = 12, radius.lg = 18. The fourth, radius.full, is a literal 9999px sentinel — the only semantic radius leaf without --nuri-px-N alias backing, because the role is "saturate to pill / circle" and the browser clamps oversized border-radius to min(width/2, height/2). A 100% percentage was tried initially but interprets relative to box dimensions, producing ellipses on rectangular elements like .nuri-tag — see N+6.1.1 post-close polish.

Cascade-invariant today — the value of --nuri-radius-md is identical across every (theme × accent) combination, so the table below renders single-Value. The dependency-driven trigger from decision 33 promotes the row to Format B automatically if a future cascade dimension splits the resolution.

The first live consumer is Button — --nuri-button-radius aliases var(--nuri-radius-md) = 12 px (the comfortable mid-scale corner softness).

Tokens

Set · core

Each row is one semantic radius leaf. Reference is the primitive it aliases (--nuri-px-N) for sm/md/lg; full is a literal with no reference. Value is the resolved CSS literal that the RN pipeline emits into build/tokens.ts under the radius namespace — all leaves emit as number post-N+6.1.1 polish (sm/md/lg as their pixel value · full as the 9999 sentinel after the px-strip in the per-component emitter). The number | string resolver widening from N+6.1 stays in place for future cascade-invariant vocabularies that may ship genuine mixed-literal leaves.

Roadmap

  • planned
    More component consumers

    Today only --nuri-button-radius consumes a radius leaf (md · 12 px). Card / Tag / InputField / Chip land in future sessions; each will alias a radius leaf inside its own @layer tokens block. radius.full will pick up consumers as soon as the first pill-shaped or circular component (Avatar · StatusPill · Spinner) ships.

  • planned
    Density dimension

    If a future data-density="compact" block lands, the same four leaves can re-map to a tighter set of primitives. The cascade-classifier auto-promotes the rows to Format B once the second cascade context exists.
