Approach

Spacing is the vocabulary gap, margin, and padding consume. Roles read as "md gap between siblings" rather than "12 pixels" — the name carries intent, the value can evolve without rewriting consumers.

The seven leaves are a T-shirt scale (2xs · xs · sm · md · lg · xl · 2xl) that aliases the --nuri-px-N primitive layer (decision 32). Spacing is asymmetric vs Sizing by design: it anchors smaller (granular between-element values 2–36 px); sizing anchors larger (element dimensions 18–90 px). Same T-shirt vocabulary, different range.

Cascade-invariant today — the value of --nuri-space-md is identical across every (theme × accent) combination, so the table below renders single-Value (no Reference Light / Reference Dark split). If a future density dimension lands, the dependency-driven trigger from decision 33 promotes the row to Format B automatically.

Tokens

Set · core

Each row is one semantic spacing leaf. Reference is the primitive it aliases (--nuri-px-N · N = literal pixel value · decision 32); Value is the resolved CSS literal that the RN pipeline emits as a JS number into build/tokens.ts under the space namespace.

Roadmap

  • planned
    Layout primitives (Stack / HStack / VStack)

    N+6.2 ships layout primitives that consume this vocabulary via a gap prop (<Stack gap="md">). Until then, components reference --nuri-space-* directly inside their @layer tokens block.

  • planned
    Density dimension

    If a future data-density="compact" block lands, the same seven leaves can re-map to a tighter set of primitives. The cascade-classifier auto-promotes the rows to Format B once the second cascade context exists.
